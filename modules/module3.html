🏃 Всё есть процесс

Зачем админу процессы?

Каждая запущенная программа — это процесс. Умение видеть и контролировать процессы — ключ к администрированию:

  • Диагностика — почему сервер тормозит? Кто ест CPU/память?
  • 🛡️ Безопасность — какие процессы запущены? Нет ли чего подозрительного?
  • ⚡ Управление — как остановить зависший сервис? Как запустить в фоне?
💡 Аналогия: Процессы — как сотрудники в офисе. Админ — менеджер, который видит, кто чем занят, кто бездельничает, и может уволить (kill) нерадивого работника.

📖 Теория: Процессы

Что такое PID?

Каждый процесс имеет уникальный номер — PID (Process ID). Также есть PPID — PID родительского процесса.

PID PPID USER COMMAND 1234 1 root /usr/sbin/nginx 1235 1234 www-data nginx: worker process
🧠 Важно: PID 1 — это systemd (или init), прародитель всех процессов. Если он умрёт — система упадёт.

Список процессов: ps

ps aux # ВСЕ процессы (a=все пользователи, u=подробно, x=без терминала) ps aux | grep nginx # найти nginx ps -ef # альтернативный формат pstree -p # дерево процессов с PID
📊 Колонки ps aux:
  • USER — владелец процесса
  • %CPU — загрузка процессора
  • %MEM — использование памяти
  • VSZ — виртуальный размер (KB)
  • RSS — реальная память (KB)
  • STAT — состояние (R=running, S=sleeping, Z=zombie)

Интерактивный монитор: top и htop

top # стандартный монитор # Горячие клавиши top: # P — сортировка по CPU # M — сортировка по памяти # k — убить процесс (ввести PID) # q — выйти htop # улучшенная версия (нужно установить: sudo apt install htop) # В htop можно кликать мышкой и использовать F1-F10
💡 Совет: htop удобнее top — цветной, с мышкой, показывает дерево процессов. Установите его сразу: sudo apt install htop

🔬 Практика: ps, top, htop

Пошаговая практика

ps aux | head -10 # первые 10 процессов ps aux | grep bash # найти все bash-процессы ps aux --sort=-%cpu | head -5 # топ-5 по CPU ps aux --sort=-%mem | head -5 # топ-5 по памяти top -n 1 -b # один снимок в текстовом режиме (для скриптов) pstree | less # дерево процессов

Установка и запуск htop

sudo apt install -y htop htop # В htop: F1 — помощь, F9 — kill, F10 — выйти

📖 Теория: Управление процессами

Сигналы: kill

Команда kill отправляет сигналы процессам. Основные сигналы:

kill PID # SIGTERM (15) — мягкое завершение (процесс может сохраниться) kill -9 PID # SIGKILL (9) — жёсткое убийство (немедленно) kill -15 PID # то же, что kill PID kill -l # список всех сигналов
⚠️ Всегда сначала пробуйте kill PID (SIGTERM). Используйте kill -9 только если процесс не реагирует — он не даёт процессу сохраниться.

Фоновые процессы: bg/fg/jobs

sleep 100 & # запустить в фоне (& в конце) jobs # показать фоновые задачи fg %1 # вернуть задачу №1 на передний план bg %1 # продолжить выполнение в фоне Ctrl+Z # приостановить текущий процесс kill %1 # убить задачу №1
🧠 Мнемоника: fg = foreground (на передний план), bg = background (в фон), & = сразу в фон.

Zombie-процессы

Если процесс завершился, но родитель не прочитал его статус — он становится zombie (статус Z в ps).

ps aux | grep Z # найти zombie-процессы
💡 Важно: Zombie-процессы не потребляют ресурсы, но засоряют таблицу процессов. Их нельзя убить — нужно перезапустить родительский процесс или дождаться его завершения.

🔬 Практика: kill, bg/fg

Пошаговая практика

sleep 300 & # запустить в фоне jobs # посмотреть номер задачи fg %1 # вернуть, потом Ctrl+C ping ya.ru > /dev/null & # запустить пинг в фоне kill %1 # убить задачу №1

Работа с PID

sleep 500 & # запустить в фоне echo $! # PID последнего фонового процесса kill $! # убить по PID ps aux | grep sleep # найти PID процесса sleep kill 1234 # убить по конкретному PID

🎯 Реальные сценарии

Сценарий 1: Сервер тормозит — кто ест CPU?

top # открыть мониторинг # Нажать P — сортировка по CPU # Найти процесс с высоким %CPU # Запомнить PID # В другом терминале: ps aux | grep <PID> # узнать, что за процесс kill <PID> # мягко завершить kill -9 <PID> # если не реагирует

Сценарий 2: Кто ест память?

ps aux --sort=-%mem | head -10 # топ-10 по памяти # Или в htop: нажать M — сортировка по памяти # Найти процесс, понять, нужен ли он # Если нет — kill

Сценарий 3: Подозрительный процесс

ps aux | grep <имя> # найти процесс ls -l /proc/<PID>/exe # показать путь к исполняемому файлу cat /proc/<PID>/cmdline # полная командная строка # Если подозрительно — kill и проверить систему
⚠️ Внимание: Если вы нашли неизвестный процесс, который потребляет много ресурсов — возможно, это майнер или бэкдор. Проверьте путь к файлу через /proc/<PID>/exe.

🔧 Troubleshooting: Если не получилось

❓ "Operation not permitted" при kill?
  • Вы не владелец процесса. Используйте sudo kill PID
  • Процесс принадлежит root — нужен sudo
❓ Процесс не убивается даже через kill -9?
  • Процесс в состоянии D (uninterruptible sleep) — ждёт I/O
  • Поможет только перезагрузка или завершение родительского процесса
  • Проверьте: ps aux | grep <PID> — посмотрите STAT
❓ "No such process" при kill?
  • Процесс уже завершился
  • Неправильный PID — проверьте через ps aux | grep <имя>
❓ Zombie-процессы в ps?
  • Zombie нельзя убить — они уже мертвы
  • Нужно перезапустить родительский процесс
  • Или дождаться, пока родитель прочитает статус
  • Найти родителя: ps -o ppid= -p <ZOMBIE_PID>
❓ htop не установлен?
  • Установите: sudo apt install htop
  • Или используйте top — он есть всегда
🛟 Универсальный совет: Если процесс "висит" и не реагирует на kill, проверьте dmesg | tail -20 — возможно, проблема на уровне ядра (I/O ошибки, нехватка памяти).

✅ Задания

Базовые задания

  • Найди процесс, занимающий больше всего памяти: ps aux --sort=-%mem | head -5
  • Запусти sleep 500 в фоне, найди его PID и уничтожь через kill
  • Приостанови текстовый редактор nano (Ctrl+Z), затем верни его через fg
  • Установи htop: sudo apt install htop и запусти
  • В htop найди процесс с высоким CPU, запомни PID, выйди и убей его через kill
  • Запусти ping ya.ru > /dev/null &, найди PID через ps, убей через kill PID
  • Посмотри дерево процессов: pstree -p | less

🧩 Задачи со звёздочкой

  • Найдите все zombie-процессы в системе: ps aux | awk '$8=="Z"'
  • Создайте скрипт, который показывает топ-5 процессов по CPU каждые 2 секунды (используйте watch)
  • Найдите процесс, который слушает порт 80: ss -tlnp | grep :80 (потребуется в модуле про сети)
  • Запустите yes > /dev/null & (загрузка CPU), найдите PID и убейте через kill -9
