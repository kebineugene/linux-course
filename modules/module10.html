Превращаем команды в программы

Зачем админу скрипты?

Bash-скрипты — это просто текстовые файлы с командами. Ты уже знаешь команды, осталось добавить логику — и рутина исчезнет. Админ без скриптов — как повар без ножа: можно, но медленно и неудобно.

  • ⚡ Автоматизация — бэкапы, мониторинг, деплой без ручного вмешательства
  • 🔄 Повторяемость — один раз написал, используешь сотни раз
  • 📝 Документирование — скрипт показывает, что и как делается
  • 🛡️ Надёжность — скрипт не забудет шаг, не опечатается
Аналогия: Если команды — это слова, то скрипты — это предложения. Ты учишься составлять инструкции, которые компьютер выполняет точно и быстро.

📖 Основы скриптов

Структура скрипта

Каждый скрипт начинается с шебанга — первой строки, которая указывает, какой интерпретатор использовать.

#!/bin/bash # Это комментарий — игнорируется интерпретатором echo "Привет, $USER!" echo "Сегодня: $(date)" echo "Рабочая директория: $(pwd)"
Шебанг (#!):
  • #!/bin/bash — использовать bash
  • #!/usr/bin/env bash — более переносимый вариант (ищет bash в PATH)
  • #!/bin/sh — POSIX-совместимый shell (менее функциональный)

Переменные и аргументы

# Обычные переменные NAME="Мир" echo "Привет, $NAME" # Аргументы командной строки echo "Имя скрипта: $0" echo "Первый аргумент: $1" echo "Второй аргумент: $2" echo "Все аргументы: $@" echo "Количество аргументов: $#" # Специальные переменные echo "PID процесса: $$" echo "Код возврата последней команды: $?"
💡 Совет: Всегда заключайте переменные в кавычки: "$NAME" вместо $NAME. Это защищает от проблем с пробелами в значениях.

🔬 Практика: первый скрипт

Создание и запуск

# 1. Создаём файл nano ~/hello.sh # 2. Пишем содержимое: #!/bin/bash echo "Привет, $USER!" echo "Сегодня: $(date)" # 3. Делаем исполняемым chmod +x ~/hello.sh # 4. Запускаем ./hello.sh
️ Важно: Если забудете chmod +x, получите ошибку "Permission denied". Альтернатива запуска без прав: bash hello.sh

🌍 Переменные окружения

Что это?

Переменные окружения хранят настройки, влияющие на поведение программ. Самые важные: PATH (где искать исполняемые файлы), HOME (домашняя директория), USER.

# Просмотр переменных echo $PATH echo $HOME echo $USER # Создание/изменение export MYVAR="hello" echo $MYVAR # Показать все переменные env printenv
💡 Сохранение: Чтобы переменная сохранялась при новых сессиях, добавьте export в ~/.bashrc или ~/.bash_profile.

Практика с PATH

# Посмотреть текущий PATH echo $PATH # Добавить папку в PATH (временно) export PATH="$PATH:/home/user/scripts" # Проверить, что работает which myscript.sh # Чтобы сделать постоянно — добавить в ~/.bashrc: echo 'export PATH="$PATH:/home/user/scripts"' >> ~/.bashrc

⭐ Best practices для скриптов

set -euo pipefail: режим строгой проверки

Эти три флага делают скрипт надёжнее. Добавляйте их в начало каждого серьёзного скрипта.

#!/bin/bash set -euo pipefail # set -e: выйти при ошибке (exit on error) # set -u: ошибка при использовании неопределённой переменной # set -o pipefail: если команда в пайпе упала — весь пайп упал echo "Этот скрипт надёжный" rm /nonexistent/file # скрипт остановится здесь
⚠️ Внимание: С set -e скрипт остановится при первой ошибке. Иногда это нежелательно (например, при проверке условий). В таких случаях используйте || true: rm file || true

Функции в bash

Функции помогают организовать код и избежать повторений.

#!/bin/bash # Определение функции log_message() { local level="$1" local message="$2" echo "[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $message" } # Использование log_message "INFO" "Скрипт запущен" log_message "ERROR" "Что-то пошло не так"
🧠 local: Переменные внутри функции лучше объявлять как local, чтобы они не конфликтовали с глобальными.

Обработка ошибок и trap

trap позволяет выполнить код при определённых событиях (ошибка, выход из скрипта, сигнал).

#!/bin/bash set -euo pipefail TMPFILE="/tmp/myapp_$$" # Очистка при выходе из скрипта cleanup() { echo "Очистка временных файлов..." rm -f "$TMPFILE" } trap cleanup EXIT # Очистка при ошибке error_handler() { echo "Ошибка на строке $1" exit 1 } trap 'error_handler $LINENO' ERR # Основная логика echo "Создаём временный файл" > "$TMPFILE" echo "Работаем..." # Если здесь ошибка — сработает trap и cleanup

Отладка скриптов

# Запуск с отладкой (показывает каждую команду перед выполнением) bash -x script.sh # Или внутри скрипта: set -x # включить отладку echo "Эта команда будет показана" set +x # выключить отладку # Отладка части скрипта: set -x # ... код для отладки ... set +x
💡 Совет: Используйте shellcheck — линтер для bash-скриптов. Он найдёт распространённые ошибки: shellcheck script.sh

Условия и циклы

if / else / elif

#!/bin/bash if [ "$1" == "admin" ]; then echo "Доступ разрешён" elif [ "$1" == "guest" ]; then echo "Ограниченный доступ" else echo "Доступ запрещён" exit 1 fi # Проверка файлов if [ -f "/etc/passwd" ]; then echo "Файл существует" fi if [ -d "/tmp" ]; then echo "Директория существует" fi if [ -r "/etc/passwd" ]; then echo "Файл читаем" fi
🧠 Операторы проверки:
  • -f file — файл существует
  • -d dir — директория существует
  • -r file — файл читаем
  • -w file — файл записываем
  • -x file — файл исполняем
  • -z "$var" — переменная пустая
  • -n "$var" — переменная не пустая

Цикл for

#!/bin/bash # Простой цикл for i in {1..5}; do echo "Шаг $i" done # Цикл по файлам for file in *.txt; do echo "Обрабатываю $file" cp "$file" "/backup/${file}.bak" done # Цикл по списку for user in alice bob charlie; do echo "Создаю пользователя $user" sudo useradd -m "$user" done

Цикл while и until

#!/bin/bash # while — пока условие истинно counter=1 while [ $counter -le 5 ]; do echo "Счётчик: $counter" ((counter++)) done # until — пока условие ложно counter=1 until [ $counter -gt 5 ]; do echo "Счётчик: $counter" ((counter++)) done # Бесконечный цикл (для мониторинга) while true; do echo "Проверка..." sleep 10 done

🔬 Практика: циклы

Скрипт бэкапа

#!/bin/bash set -euo pipefail # Создаём директорию для бэкапа DIR="/tmp/backup_$(date +%Y%m%d_%H%M%S)" mkdir -p "$DIR" echo "Начинаю бэкап в $DIR" # Копируем все .txt файлы из домашней папки for file in ~/*.txt; do if [ -f "$file" ]; then cp "$file" "$DIR/" echo "Скопирован: $(basename $file)" fi done echo "Бэкап завершён в $DIR" echo "Размер: $(du -sh $DIR | cut -f1)"

Скрипт мониторинга

#!/bin/bash # Проверяем сервис каждые 30 секунд while true; do if systemctl is-active --quiet nginx; then echo "✅ nginx работает" else echo "❌ nginx УПАЛ! Перезапускаю..." sudo systemctl restart nginx fi sleep 30 done

📦 Git для админа: храним скрипты и конфиги

Зачем?

Git — не только для программистов. Администраторы хранят в репозитории скрипты, конфигурационные файлы, заметки. Это даёт историю изменений и бекап.

Базовый рабочий процесс

# Инициализация репозитория cd ~/scripts git init # Первый коммит git add . git commit -m "Первый коммит: мои скрипты" # После изменений git add backup.sh git commit -m "Добавил ротацию в backup.sh" # История git log --oneline git log --oneline --graph --all

Ветки и слияния

# Создать ветку git branch feature-nginx-config git checkout feature-nginx-config # Или одной командой: git checkout -b feature-nginx-config # Внести изменения, закоммитить git add nginx.conf git commit -m "Настройка nginx для production" # Вернуться в main и слить изменения git checkout main git merge feature-nginx-config # Удалить ветку git branch -d feature-nginx-config

.gitignore для админа

Исключайте чувствительные файлы из репозитория:

# .gitignore *.password *.key *.secret .env config.local.php backups/ *.log
🚨 Критично: Никогда не коммитьте пароли, ключи SSH, токены API! Даже если потом удалите — они останутся в истории Git.

Удалённый репозиторий

# Добавить remote git remote add origin https://github.com/user/my-scripts.git # Отправить git push -u origin main # Получить обновления git pull origin main # Клонировать на другой сервер git clone https://github.com/user/my-scripts.git
💡 Совет: Храните конфиги из /etc в отдельном репозитории с помощью etckeeper или вручную.

Планировщик cron

Запуск скриптов по расписанию

# Редактировать cron текущего пользователя crontab -e # Синтаксис: минута час день месяц день_недели команда # * = любое значение # Примеры: # Каждый день в 2:30 ночи 30 2 * * * /home/user/backup.sh # Каждые 15 минут */15 * * * * /home/user/check.sh # Каждый понедельник в 9:00 0 9 * * 1 /home/user/weekly-report.sh # 1-е число каждого месяца в полночь 0 0 1 * * /home/user/monthly-backup.sh
🧠 Формат cron:
минута(0-59) час(0-23) день(1-31) месяц(1-12) день_недели(0-7)

Специальные строки:
  • @reboot — при загрузке системы
  • @daily — каждый день (0 0 * * *)
  • @weekly — каждую неделю (0 0 * * 0)
  • @monthly — каждый месяц (0 0 1 * *)

Важные моменты cron

# Просмотреть задачи crontab -l # Удалить все задачи crontab -r # Логи cron sudo journalctl -u cron --since today # Перенаправить вывод в лог 30 2 * * * /home/user/backup.sh >> /var/log/backup.log 2>&1
️ Окружение cron: Cron запускает скрипты с минимальным окружением. Нет PATH, нет переменных пользователя. Всегда используйте полные пути к командам: /usr/bin/curl вместо curl.

🎯 Реальные сценарии

Сценарий 1: Скрипт бэкапа с логированием

#!/bin/bash set -euo pipefail LOG="/var/log/backup.log" BACKUP_DIR="/backup/$(date +%Y%m%d)" log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG" } log "Начинаю бэкап" # Создаём директорию mkdir -p "$BACKUP_DIR" # Бэкап базы данных log "Бэкап MySQL" mysqldump -u root --all-databases | gzip > "$BACKUP_DIR/db.sql.gz" # Бэкап файлов log "Бэкап /var/www" tar -czf "$BACKUP_DIR/www.tar.gz" /var/www # Удаляем старые бэкапы (старше 30 дней) log "Удаление старых бэкапов" find /backup -type f -mtime +30 -delete log "Бэкап завершён"

Сценарий 2: Скрипт мониторинга с алертами

#!/bin/bash # Проверяем место на диске USAGE=$(df / | awk 'NR==2 {print $5}' | sed 's/%//') if [ "$USAGE" -gt 90 ]; then echo "⚠️ Диск заполнен на $USAGE%!" | wall curl -s -X POST "https://api.telegram.org/bot$TOKEN/sendMessage" \ -d chat_id="$CHAT_ID" \ -d text="⚠️ Диск заполнен на ${USAGE}%" > /dev/null fi # Проверяем сервисы for SERVICE in nginx mysql ssh; do if ! systemctl is-active --quiet "$SERVICE"; then echo " $SERVICE упал!" | wall sudo systemctl restart "$SERVICE" fi done

Сценарий 3: Автоматизация деплоя

#!/bin/bash set -euo pipefail APP_DIR="/var/www/myapp" BACKUP_DIR="/backup/myapp_$(date +%Y%m%d_%H%M%S)" echo "Начинаю деплой..." # Бэкап текущей версии echo "Создаю бэкап" cp -r "$APP_DIR" "$BACKUP_DIR" # Получаем новую версию echo "Pull из Git" cd "$APP_DIR" git pull origin main # Устанавливаем зависимости echo "Установка зависимостей" npm install --production # Перезапускаем сервис echo "Перезапуск сервиса" sudo systemctl restart myapp echo "Деплой завершён"

🔧 Troubleshooting: Если не получилось

❓ "Permission denied" при запуске скрипта
  • Скрипт не исполняемый: chmod +x script.sh
  • Или запускайте через bash: bash script.sh
❓ "command not found" в скрипте, но работает в терминале
  • Проблема с PATH — cron имеет минимальное окружение
  • Используйте полные пути: /usr/bin/curl вместо curl
  • Или добавьте PATH в начало скрипта: PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
❓ Скрипт работает вручную, но не в cron
  • Проверьте PATH (см. выше)
  • Проверьте права на файл: ls -l script.sh
  • Проверьте, что cron запущен: systemctl status cron
  • Смотрите логи cron: sudo journalctl -u cron --since "1 hour ago"
  • Добавьте вывод в лог: * * * * * /script.sh >> /tmp/cron.log 2>&1
❓ "bash: syntax error near unexpected token"
  • Проверьте синтаксис: bash -n script.sh (проверка без запуска)
  • Используйте shellcheck script.sh для поиска ошибок
  • Проверьте кавычки и скобки
  • Запустите с отладкой: bash -x script.sh
❓ Переменная пустая внутри скрипта
  • Проверьте, что переменная экспортирована: export MYVAR=value
  • Используйте set -u для обнаружения неопределённых переменных
  • Проверьте опечатки в имени переменной
🛟 Универсальный совет: Если скрипт не работает — добавьте set -x в начало. Это покажет каждую команду перед выполнением. Вы сразу увидите, где проблема.

✅ Задания

Базовые задания

  • Напиши скрипт, который выводит «Доброе утро», если время до 12, иначе «Добрый день» (используй date +%H)
  • Создай скрипт, который архивирует папку /var/log в /tmp/logs.tar.gz с датой в имени файла
  • Инициализируй Git-репозиторий в папке со скриптами, сделай первый коммит
  • Выведи переменную PATH и добавь в неё новую папку (временно через export)
  • Напиши скрипт с функцией log_message(), которая выводит timestamp и сообщение
  • Создай скрипт с set -euo pipefail и trap для очистки временных файлов
  • Напиши цикл for, который создаёт 5 файлов: file1.txt, file2.txt, ..., file5.txt
  • Добавь скрипт архивации в cron на выполнение каждую пятницу в 18:00

🧩 Задачи со звёздочкой

  • Напишите скрипт, который принимает аргументы через getopts: -d (директория), -o (output файл), -h (help)
  • Создайте скрипт мониторинга, который проверяет: место на диске, загрузку CPU, статус 3 сервисов — и отправляет сводный алерт в Telegram, если что-то не в порядке
  • Напишите функцию retry(), которая выполняет команду до 3 раз с задержкой 5 секунд, если команда падает
  • Создайте Git-репозиторий с ветками main и develop, внесите изменения в develop, слейте в main через merge
