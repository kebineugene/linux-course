Добро пожаловать!

Почему терминал?

Представь, что ты хирург:

  • GUI — операция через видеоэкран (удобно, но ограничено)
  • Терминал — работа напрямую скальпелем (быстро, точно, полный контроль)

Главное правило: В Linux всё есть файл. Научишься работать с файлами — научишься управлять системой.

💡 Зачем это нужно админу?
  • 90% серверов Linux управляются только через терминал
  • GUI на сервере часто не установлен — экономит ресурсы
  • Терминал позволяет автоматизировать любые задачи через скрипты

💻 Быстрый старт: WSL для Windows

Установка Linux на Windows

Откройте PowerShell от имени администратора и выполните:

wsl --install -d Ubuntu

После перезагрузки откройте Ubuntu из меню Пуск и создайте пользователя. Это ваш Linux-терминал для практики.

Mac/Linux: просто откройте встроенный терминал (Terminal.app или Konsole/GNOME Terminal).

⚠️ Важно: Если WSL не устанавливается, включите виртуализацию в BIOS (Intel VT-x / AMD-V). Без неё Linux не запустится.

📖 Теория: Файловая система

Дерево каталогов

В Linux есть один корень /. Всё начинается с него:

/
 ├── bin/          # Основные команды (ls, cp, bash)
 ├── boot/         # Файлы загрузчика
 ├── dev/          # Устройства (диски, терминалы)
 ├── etc/          # Конфигурационные файлы системы
 ├── home/         # Домашние папки пользователей
 │   └── user/     # Ваша папка (~)
 ├── lib/          # Библиотеки программ
 ├── media/        # Съёмные носители (флешки)
 ├── opt/          # Дополнительное ПО
 ├── proc/         # Информация о процессах (виртуальная ФС)
 ├── root/         # Домашняя папка root
 ├── run/          # Временные данные запущенных сервисов
 ├── sbin/         # Системные команды (требуют root)
 ├── tmp/          # Временные файлы (очищается при перезагрузке)
 ├── usr/          # Пользовательские программы и данные
 └── var/          # Логи, кэш, базы данных

Абсолютные и относительные пути

Абсолютный путь — начинается с корня /:

/home/user/documents/file.txt

Относительный путь — от текущей папки:

documents/file.txt
⚡ Правило: Если путь начинается с / — это от корня. Если нет — от текущей папки.

cd documents → /home/user/documents (если вы в /home/user)
cd /documents → /documents (в корне, которого скорее всего нет!)

🔬 Практика: Навигация

Команда pwd — Где я?

pwd

Зачем: Чтобы понять, в какой папке вы находитесь, прежде чем что-то удалять или создавать.

Команда ls — Что вокруг?

ls # просто список ls -la # ВСЁ: скрытые файлы, права, размеры, дата ls -lah # Размеры в удобном формате (КБ/МБ/ГБ) ls -l *.txt # Только .txt файлы
💡 Подсказка: Флаги можно комбинировать: -l (подробно) + -a (все, включая скрытые) + -h (human-readable).

Команда cd — Куда пойти?

cd /etc # перейти в /etc cd .. # шаг назад (в родительскую папку) cd ~ # домой (в /home/user) cd # тоже домой cd - # вернуться в предыдущую папку

Пошаговая практика

Выполните команды по очереди и наблюдайте за результатом:

pwd # где я? ls -la # что в папке? cd ~ # домой mkdir practice # создать папку cd practice # зайти в неё pwd # проверить, где я

📹 Демонстрация: навигация в терминале

Повтори команды из видео в своём терминале!

💡 Совет: Используй Tab для автодополнения имён файлов и папок — это экономит время и защищает от опечаток.

📖 Теория: Работа с файлами

Создание файлов

touch file.txt # пустой файл echo "Привет" > file.txt # создать с текстом echo "Ещё" >> file.txt # ДОБАВИТЬ в конец
⚠️ Критично: > перезаписывает файл целиком! >> дописывает в конец.

Если случайно напишете echo "new" > important.txt — старый контент исчезнет навсегда!

Копирование, перемещение, удаление

cp file1.txt file1_copy.txt # копировать файл cp -r dir1/ dir2/ # копировать папку (рекурсивно) mv file.txt newname.txt # переименовать / переместить rm file.txt # удалить файл rm -r dir/ # удалить папку со всем содержимым
В Linux НЕТ корзины! Команда rm удаляет файлы навсегда, без возможности восстановления.

Совет: Сначала используйте mv file.txt ~/.trash/ вместо rm, пока не набьёте руку.

Просмотр содержимого

cat file.txt # весь файл сразу less file.txt # постранично (q — выйти) head -10 file.txt # первые 10 строк tail -10 file.txt # последние 10 строк tail -f /var/log/syslog # следить за логом в реальном времени
Зачем tail -f? Админы используют эту команду, чтобы видеть логи сервиса в реальном времени, пока он работает.

🔬 Практика: Файлы

Пошаговая практика

mkdir ~/practice cd ~/practice echo "Учу Linux" > notes.txt echo "Это круто" >> notes.txt cat notes.txt cp notes.txt notes_backup.txt mv notes_backup.txt notes_old.txt rm notes_old.txt ls -la

📹 Демонстрация: создание и управление файлами

Обрати внимание на разницу между > и >>!

️ Запомни: > = перезаписать (осторожно!), >> = добавить в конец (безопасно).

🔄 Перенаправления и пайпы

Перенаправление вывода

echo "Hello" > file.txt # записать (перезаписать!) echo "World" >> file.txt # дописать в конец ls /fake 2> error.log # ошибки в файл ls /fake 2>&1 # ошибки туда же, куда обычный вывод

Пайпы (конвейер) |

Символ | передаёт вывод одной команды на вход другой:

ls -la | grep ".txt" # найти файлы .txt cat /etc/passwd | wc -l # посчитать строки ps aux | grep nginx | wc -l # сколько процессов nginx работает
💡 Зачем? Пайпы — это суперсила Linux. Можно комбинировать простые команды в мощные конвейеры обработки данных.

📹 Демонстрация: пайпы и перенаправления

Пайпы позволяют строить мощные конвейеры из простых команд

🧠 Мнемоника: cut → sort → uniq → sort → head — стандартный конвейер для подсчёта повторов.

📦 Архивация и сжатие

tar — упаковка файлов

В Linux архивирование и сжатие — это две операции. tar объединяет файлы, gzip сжимает.

# Создать архив и сжать: tar -czf archive.tar.gz папка/ # Распаковать: tar -xzf archive.tar.gz # Создать несжатый tar: tar -cf archive.tar папка/ # Просмотреть содержимое без распаковки: tar -tzf archive.tar.gz
🧠 Запомни мнемонику: c = create, x = extract, t = list, z = gzip, f = file.

Практика

cd ~ tar -czf practice_backup.tar.gz practice/ ls -lh practice_backup.tar.gz mkdir /tmp/restore tar -xzf practice_backup.tar.gz -C /tmp/restore/ ls /tmp/restore/

Демонстрация: архивация с tar

tar -czf создаёт архив, tar -tzf показывает содержимое без распаковки

💡 Совет: Добавляй дату в имя архива: backup-$(date +%Y%m%d).tar.gz — так легко найти нужный бэкап.

✏️ Текстовый редактор nano

Зачем нужен редактор?

Чтобы править конфиги и писать скрипты, нужно уметь редактировать файлы прямо в терминале. nano — самый простой редактор для начала.

nano file.txt # открыть (или создать) # Внутри nano: # Ctrl+O — сохранить (WriteOut) # Ctrl+X — выйти # Ctrl+K — вырезать строку # Ctrl+U — вставить строку # Ctrl+W — поиск

📹 Демонстрация: редактирование в nano

nano — самый простой редактор для новичков

🧠 Запомни: Ctrl+O = сохранить, Ctrl+X = выйти. Подсказки всегда внизу экрана!

🧠 Мини-курс vim (экстренный выход)

Зачем?

На многих минимальных серверах nano отсутствует, а vim (или vi) — есть всегда. Научись трём вещам, чтобы не паниковать.

vim file.txt # Нажатие i — переход в режим вставки (как в блокноте) # После правки: # Esc — выйти из режима вставки # :wq — сохранить и выйти # :q! — выйти без сохранения
️ Если vim "застрял" и вы не знаете, что делать — нажмите Esc, затем наберите :q! и Enter. Это аварийный выход.

📹 Демонстрация: как выйти из Vim

Запомни: Esc → :q! → Enter = аварийный выход без сохранения

🆘 Экстренный выход из Vim:
  1. Нажми Esc (несколько раз)
  2. Набери :q!
  3. Нажми Enter

📖 Теория: Поиск

Поиск файлов: find

find /etc -name "*.conf" # найти .conf в /etc find / -name "nginx.conf" 2>/dev/null # во всей системе (ошибки скрыть) find / -size +100M # файлы больше 100 МБ find ~ -type f -mtime -7 # файлы, изменённые за последние 7 дней

Поиск текста: grep

grep "error" /var/log/syslog # найти "error" grep -r "root" /etc/ # во всех файлах рекурсивно grep -i "ERROR" log.txt # без учёта регистра grep -v "error" log.txt # строки БЕЗ "error" grep -n "error" log.txt # с номерами строк
💡 Комбинация: find / -name "*.log" 2>/dev/null | xargs grep "error" — найти слово "error" во всех .log файлах системы.

🔬 Практика: Поиск

Пошаговая практика

find ~ -name "*.txt" grep -r "root" /etc/ 2>/dev/null | head -5 head -5 /etc/passwd tail -5 /etc/passwd ps aux | grep bash | grep -v grep # найти процессы bash (исключив сам grep)

📹 Демонстрация: поиск файлов и текста

find ищет файлы, grep ищет текст внутри файлов

💡 Совет: Добавляй 2>/dev/null к find, чтобы скрыть ошибки "Permission denied" при поиске по всей системе.

📚 Справка: man и --help

Как узнать больше о любой команде?

В Linux встроена полная документация. Вам не нужен интернет — всё уже есть в системе.

man ls # полная справка по команде ls man find # справка по find q # выйти из man (нажмите q) ls --help # краткая справка (быстрее, чем man) find --help | less # длинная справка с прокруткой info ls # альтернатива man (более подробная)
🎯 Навык админа: Умение читать man важнее, чем запоминать все флаги. Если вы знаете, как найти ответ — вы уже наполовину админ.

🎯 Реальные сценарии

Сценарий 1: "Не могу найти конфиг nginx"

# 1. Ищем файл конфига sudo find / -name "nginx.conf" 2>/dev/null # 2. Нашли: /etc/nginx/nginx.conf # 3. Смотрим содержимое sudo cat /etc/nginx/nginx.conf # 4. Или сразу открываем в редакторе sudo nano /etc/nginx/nginx.conf

Сценарий 2: "Нужно найти все логи старше 30 дней"

# 1. Находим все .log файлы старше 30 дней sudo find /var/log -name "*.log" -mtime +30 # 2. Смотрим размер sudo find /var/log -name "*.log" -mtime +30 -exec ls -lh {} \; # 3. Удаляем (осторожно!) sudo find /var/log -name "*.log" -mtime +30 -delete

Сценарий 3: "Нужно сделать бэкап папки перед изменениями"

# 1. Создаём бэкап с датой tar -czf /tmp/nginx-backup-$(date +%Y%m%d).tar.gz /etc/nginx/ # 2. Проверяем бэкап ls -lh /tmp/nginx-backup-*.tar.gz tar -tzf /tmp/nginx-backup-$(date +%Y%m%d).tar.gz | head -10 # 3. Теперь можно смело менять конфиги sudo nano /etc/nginx/nginx.conf

🔧 Troubleshooting: Если не получилось

❓ "No such file or directory"
  • Проверьте путь через pwd и ls
  • Опечатка в имени? Используйте Tab для автодополнения
  • Файл в другой папке? Используйте абсолютный путь
❓ "Permission denied"
  • У вас нет прав на файл/папку
  • Попробуйте sudo перед командой (если вы админ)
  • Или проверьте владельца: ls -la file.txt
❓ "Command not found"
  • Опечатка в команде? Проверьте man
  • Программа не установлена? sudo apt install <имя>
  • Нужен полный путь? /usr/bin/ls
"Is a directory"
  • Вы пытаетесь работать с папкой как с файлом
  • Для папок нужен флаг -r (рекурсивно): cp -r dir1/ dir2/
❓ vim "завис" и не выходит
  • Нажмите Esc (возможно, несколько раз)
  • Наберите :q! и Enter — аварийный выход
  • Если совсем никак — Ctrl+C, потом :q!
🛟 Универсальный совет: Если команда не работает — добавьте 2>&1 | head -20 в конец, чтобы увидеть первые 20 строк ошибок.
